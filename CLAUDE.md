# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start          # local dev server with hot reload
npm run build      # generate static output to ./build
npm run serve      # serve the built site locally
npm run clear      # clear Docusaurus cache (run when builds behave oddly)
```

Deploy to GitHub Pages:
```bash
GIT_USER=<github-username> npm run deploy
```

No test suite exists — the linting tools below are the quality gates.

## Linting

Three complementary linters run on Markdown content:

**Vale** — deterministic style rules using the Microsoft style pack:
```bash
vale docs/ blog/
```
Config: `.vale.ini` (styles in `lint-styles/Microsoft/`)

**textlint** — prose quality (passive voice, weasel words, clichés, misspellings):
```bash
npx textlint docs/ blog/
```
Config: `.textlintrc`

**VectorLint** — AI-augmented evaluation using the style contract in `VECTORLINT.md`. The style contract is prepended as the system prompt to every VectorLint call. The `notebooks/vectorlint-pipeline-demo-anthropic.ipynb` notebook shows exactly how the API call is constructed and how scores are computed.

## Architecture

**Docusaurus v3** with two presets:
- `@docusaurus/preset-classic` — docs, blog, theme
- `redocusaurus` — renders OpenAPI specs at `/api/*`; scans the `openapi/` directory automatically

**Content areas:**
- `docs/my-docs/` — portfolio content (resume, writing samples, JVN chatbot docs)
- `blog/` — blog posts; `blog/authors.yml` and `blog/tags.yml` define shared metadata
- `openapi/jvn-chatbot/` — OpenAPI spec rendered at `/api/jvn-chatbot`
- `src/pages/` — standalone React pages outside the docs/blog routing
- `src/components/HomepageFeatures/` — homepage feature cards

**Sidebar** (`sidebars.js`) is manually configured — not autogenerated. Add new `docs/my-docs/` pages there explicitly or they won't appear in nav.

**Style contract** (`VECTORLINT.md`) is the global writing style guide. It defines voice, tone, structure, list, procedure, and terminology rules. Treat it as authoritative for all content in `docs/` and `blog/`. When drafting or editing documentation, apply these rules directly — active voice, present tense, sentence-case headings, no passive constructions, no weasel words.

**CI/CD:** `.github/workflows/static.yml` builds and deploys to GitHub Pages on every push to `main` via `npm ci && npm run build`.

**Notebooks** (`notebooks/`) demonstrate the VectorLint AI pipeline using the Anthropic SDK. They require `ANTHROPIC_API_KEY` to run and use `claude-sonnet-4-6`.
